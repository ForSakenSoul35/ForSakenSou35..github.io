JavaScript异步解决方案

发表于 2019-09-02 | 分类于 JavaScript

六种异步解决方案

回调函数

回调函数是异步操作最基本的方法
但是回调函数有一个致命的弱点，就是容易写出回调地狱。 假设多个请求存在依赖性，就有可能出现回调地狱。
回调函数的优点是简单，容易理解以及实现。
缺点是不利于代码的阅读和维护，各个部分之间高度耦合，使得程序之间结构混乱，难以追踪，每个任务都只能指定一个回调函数。
此外不能使用try …catch捕获错误，不能直接return

事件监听

这种情况下，异步任务的执行，不取决于代码的顺序，而是取决于某个事件是否发生

发布订阅

假定存在一个信号中心，某个任务执行完成，就向信号中心发布一个信号，其他任务可以
向信号中心 订阅 这个信号，从而知道什么时候自己可以开始执行，这就叫做 发布订阅模式

Promise/A+

Ajax的简单实现

为了我们的代码更加具有可读性和维护性 需要将数据请求以及数据处理明确的区分开来

如果浏览器已经支持了原生的Promise对象，浏览器的js引擎中就已经有了Promise队列，这样就可以利用Promise将任务放在它的队列中

Promise 基础知识

  1. Promise对象有三种状态
  • pending 等待中 或者进行中 表示还没有结果
  • resolved 已经完成 表示得到了我们想要的结果 可以继续往下执行
  • rejected 也表示得到结果 但是由于结果并非我们所愿 因此拒绝执行

这三种状态 不受外界影响 ，且状态只能从pending->resolved 或者 pending-> rejected，并且不可逆。
在Promise对象的构造函数中，将一个函数作为第一个参数，而这个参数，就是用来处理Promise 的状态变化

  1. Promise对象中的then方法

可以接收构造函数中处理的状态变化，并分别对应执行。
then方法有两个参数，第一个函数接收resolved状态的执行，第二个参数接受reject状态的执行

then方法的执行结果也是返回一个Promise对象。因此可以进行then的链式调用，这也是解决回调地狱的主要方式。

  1. Promise的数据传递

Promise的基本语法

  • Promise实例必须实现then方法
  • then（）方法可以接收两个函数作为参数
  • then（） 方法返回一个Promise对象

Promise的含义

Promise是异步编程的一种解决方案
所谓Promise，简单来说，就是一个容器，里面保存着某个未来才会结束的事件。
Promise是一个对象。
提供统一的API，各种异步操作都可以用同样的方法来进行处理。

Promise的特点

  1. 对象的状态不受外界影响。
  2. 一旦状态改变，就不会再变

Promise基本用法

ES6规定，Promise对象是一个构造函数，用来生成Promise实例。


Promise构造函数 接收一个函数作为参数，该函数的两个参数分别为resolve和reject。它们是两个函数，由JS引擎提供。
resolve函数的作用，将Promise对象的状态从未完成变为成功，在异步操作成功时调用，并将异步操作的结果，作为参数传递出去（resolve(value)）
reject函数的作用，是将Promise对象的状态从未完成变为失败，在异步操作失败时调用，并将异步操作报出的错误，作为参数传递出去
Promise实例生成之后，可以用then方法分别指定resolved和rejected的回调函数。

then方法接收两个回调函数。第一个回调参数是Promise对象的状态变为resolved时调用，第二个回调函数是Promise对象的状态变为reject时调用

timeout方法 返回一个Promise实例，表示一段时间以后才会发生的结果。
过了指定的时间以后，Promise实例的状态未resolved，就会触发then方法绑定的回调函数。

Promise新建后就会立即执行。

Promise新建之后立即执行，首先输出的Promise，然后then方法指定的回调函数，会在当前脚本所有同步任务执行完参会执行，所以resolved最后执行（ 可参考event lop）

reject函数的参数通常是Error对象的实例，表示抛出的错误。resolve函数的参数除了正常的参数，还可能使另外一个Promise实例

调用resolve之后，后面的代码还是会继续执行，并且会首先打印出来，这是因为立即resolve的Promise是在本轮事件循环的末尾执行，总是晚于本轮循环的同步任务。

调用resolve和reject以后，Promise的使命就完成，后续操作应该放到then方法中，而不应该直接写在resolve或者reject后面。所以应该在它们前面加入 return语句，这样就不会出现意外

then方法

Promise的实例具有then方法，也就是说then方法是定义在原型对象Promise.prototype上。
他的作用是为Promise实例添加状态改变时的回调函数，
then方法有两个参数。第一个是resolved状态的回调函数，第二个参数（可选） 是rejected状态的回调函数
then方法返回的是一个新的Promise实例，因此可以采用链式写法。即then方法后再调用另一个then方法。

采用链式调用的then，依次指定了两个回调函数，第一个回调函数 完成以后，会将返回的结果作为参数，传入第二个回调函数。

采用链式的then，可以指定一组按照次序调用的回调函数，这时，前一个回调函数，有可能返回的还是一个Promise对象，即有异步操作，这时，后一个回调函数就会等待该Promise对象的状态发生改变，才会被调用

Promise.prototype.catch

是 .thne(null,rejection)或者.then(undefined,rejection)的别名，用于执行发生错误时的回调函数

getJSON返回一个Promise对象，如果Promise的对象状态变为resolve，则会调用then方法指定的回调函数，如果状态为rejected，就会调用catch指定的回调函数。
另外如果then方法指定的回调函数，如果在运行中发生错误，catch方法也能捕获到。

Promise对象的错误 具有冒泡性质，会一直向后传递，直到被捕获为止，也就是错误总会被下个catch捕获
一般来说，不要在then方法中定义Reject状态的回调函数（即then方法的第二个参数），总是使用catch方法，因为能捕获到then的方法抛出的错误。
为什么？ 因为catch可以捕获之前发生的错误，也更接近同步try-catch的写法
与传统的try..catch不同的是，如果没有使用catch方法指定错误处理的回调函数，Promise对象抛出的错误不会传递到外层代码，即不会有任何反应

Promise会吃掉错误，setTimeout还是会执行
一般总是建议 Promise对象之后要跟catch方法，这样可以处理Promise内部发生的错误
catch方法返回的还是一个Promise对象，因此还可以继续调用then方法

Promise.prototype.finally

finally用于指定不管Promise对象最后状态如何，都会执行的操作。
该方法是 ES2018引入的。

finally的回调函数不接受任何参数，这意味着没有办法直到，前面的Promise状态到底是怎么样，说明，finally
里面的方法应该不依赖Promise的执行结果。

Promise.all()

用于将多个Promise实例包装成一个新的Promise实例
接收一个数组作为参数。可以不是数组，但是一定要有Iterator接口，返回必须是Promise实例
新的Promise实例由所有的Promise实例决定 只要有一个Promise实例的状态变为rejected，那么新的Promise实例状态就为rejected。
如果作为参数的Promise实例定义了自己的catch，发生错误时，就不会触发Promise.all自己的catch方法
如果作为参数的Promise实例没有自己的catch，发生错误时，就会调用Promise.all自己的catch方法

Promise.race()

Promise.race将多个Promise实例包装成一个新的Promise实例.
接收一个数组作为参数
